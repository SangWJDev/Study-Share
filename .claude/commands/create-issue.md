# 이슈 생성

사용자가 구현할 기능에 대한 GitHub Issue를 생성합니다.

## 입력 받을 정보

사용자에게 아래 정보를 질문하세요:
1. 기능 제목 (예: 회원가입 API 구현)
2. 기능 설명 (간단한 설명)
3. 세부 작업 목록 (체크박스로 만들 항목들)

## 이슈 템플릿

```markdown
## 📋 기능 설명
{기능 설명}

## ✅ 세부 작업
- [ ] 작업1
- [ ] 작업2
- [ ] 작업3

## 🔗 관련 정보
- 관련 ERD 테이블:
- API 엔드포인트:

## 📝 참고 사항
```

## 실행 방법

`gh issue create` 명령어를 사용하여 이슈를 생성하세요.

```bash
gh issue create --title "제목" --body "본문"
```

## 주의사항

- 이슈 번호를 사용자에게 알려주세요 (브랜치명에 사용)
- 브랜치명 제안: `feature/#이슈번호-간단한설명`
