# PR 생성 및 코드 리뷰

기능 구현 완료 후 PR을 생성하고 코드 리뷰를 수행합니다.

## 수행 작업

### 1. 브랜치 및 이슈 확인
```bash
git branch --show-current
git log main..HEAD --oneline
```

### 2. 전체 변경 사항 확인
```bash
git diff main...HEAD
```

### 3. 코드 리뷰 수행

**Claude는 코드를 직접 수정하지 않습니다.**

아래 관점에서 피드백을 제공하세요:

#### 체크리스트
- [ ] 네이밍 컨벤션 준수 여부
- [ ] 불필요한 코드/주석 존재 여부
- [ ] 예외 처리 적절성
- [ ] N+1 쿼리 가능성
- [ ] 보안 취약점 (SQL Injection, XSS 등)
- [ ] 트랜잭션 범위 적절성
- [ ] DTO/Entity 분리 여부
- [ ] 테스트 코드 존재 여부

#### 피드백 형식
```
## 📝 코드 리뷰

### ✅ 잘한 점
- 

### ⚠️ 개선 필요
- 파일명: 
  - 문제: 
  - 제안: 

### 💡 추가 제안
- 

### ❓ 질문
- 이 부분은 왜 이렇게 구현했나요?
```

### 4. PR 생성

리뷰 완료 후 사용자 동의를 받고 PR을 생성합니다.

```bash
gh pr create --title "제목" --body "본문"
```

#### PR 템플릿

```markdown
## 📋 관련 이슈
- closes #{이슈번호}

## 🔧 변경 사항
- 

## ✅ 체크리스트
- [ ] 코드 리뷰 피드백 반영
- [ ] 테스트 통과
- [ ] 문서 업데이트 (필요시)

## 📸 스크린샷 (선택)

## 💬 기타
```

### 5. 피드백 반영 확인

사용자가 피드백을 반영하면 다시 확인 후 PR 승인 의견을 남깁니다.

## 주의사항

- **코드 직접 수정 금지** - 피드백만 제공
- 사용자가 스스로 수정하도록 유도
- 수정 방법을 힌트로만 제공 (전체 코드 X)
